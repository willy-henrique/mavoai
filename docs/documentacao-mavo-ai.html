01 · O que é

O que é o Mavo AI?

O Mavo AI é uma plataforma de suporte técnico inteligente desenvolvida especificamente para o ecossistema AUGE ERP. Ele funciona como um agente autônomo de atendimento que opera 24 horas por dia, 7 dias por semana, diretamente no WhatsApp dos clientes.

Quando um cliente entra em contato relatando um problema — seja com nota fiscal, TEF, estoque, banco de dados ou qualquer módulo do AUGE — o sistema assume o atendimento, classifica o problema, coleta as evidências necessárias e tenta resolver de forma totalmente autônoma, sem precisar de um técnico humano na maioria dos casos.

💡 Em termos simples: é como ter um técnico sênior do AUGE ERP disponível 24h no WhatsApp de cada cliente, que nunca se cansa, nunca esquece um procedimento e aprende com cada atendimento.

Fluxo de Atendimento
04 · Como funciona

Fluxo completo de um atendimento

Cada mensagem recebida passa por um orquestrador inteligente que decide automaticamente qual fase executar com base no histórico e estado da conversa. O fluxo tem 5 fases distintas:

1

Recepção da Mensagem

A mensagem chega via plataforma integrada (WillTalk, MTalk, ou API direta). O sistema valida a organização, aplica rate limiting e identifica o cliente pelo telefone. Texto, imagens e áudios transcritos são processados.

2

Triagem Automática — Classificação do Chamado

A IA analisa a mensagem, identifica o módulo afetado (Fiscal, TEF, Estoque, Banco de Dados...) e define prioridade: Baixa / Média / Alta / Crítica e severidade S1 a S4. Direciona para a fila de atendimento correta automaticamente.

3

Investigação — Coleta de Evidências

Se as informações forem insuficientes, a IA faz uma pergunta objetiva por vez para coletar: mensagem de erro exata, módulo afetado, impacto (total ou parcial). Aceita prints de tela enviados pelo cliente e analisa as imagens automaticamente com visão computacional.

⚡

Resolução Autônoma — Até 3 Tentativas

Com as evidências em mãos, a IA busca casos similares na base de conhecimento semântica, consulta os playbooks técnicos do AUGE ERP e gera uma solução passo a passo. Se a primeira tentativa não resolver, tenta uma causa raiz diferente (máx. 3 tentativas). Cada tentativa é verificada com o cliente.

H

Handoff Humano — Quando Necessário

Se após 3 tentativas o problema persistir, se o cliente pedir explicitamente um técnico, ou se a situação for crítica (produção parada, SEFAZ bloqueada), o sistema gera automaticamente um resumo técnico estruturado com todo o diagnóstico e transfere para o técnico humano sem perda de contexto.

🔄 Contexto contínuo: o sistema lembra de tudo que foi dito na conversa. Nunca repete a mesma pergunta. Se o cliente diz "funcionou, mas agora tenho outro erro", a IA entende o novo problema e inicia uma nova investigação automaticamente.

Base de Conhecimento & Módulos
05 · Base de conhecimento

O "Cérebro" — Base de Conhecimento AUGE ERP

O coração do sistema é uma base de conhecimento estruturada com 27 módulos técnicos do AUGE ERP, playbooks de diagnóstico, casos reais resolvidos e procedimentos passo a passo. Cada módulo contém causas raiz, soluções, erros comuns e checklists de diagnóstico.

Módulos cobertos:

Fiscal NF-e (Modelo 55) NFC-e (Modelo 65) SAT CF-e SPED Fiscal Tributação / CST / NCM Sintegra Certificado Digital Reforma Tributária PDV / Caixa TEF / PIX / SiTef Balança Perfil de Movimento Vendas Retaguarda Estoque Contagem de Estoque Compras / Entradas Financeiro Financeiro Avançado Cadastro de Produto Clientes Fornecedores / Mobile Hardware / PinPad Banco de Dados Instalação / Atualização Filiais e Usuários Geral
06 · Busca semântica

Como a IA "lembra" de casos resolvidos

Além dos módulos estáticos, o sistema aprende continuamente. Cada atendimento resolvido é convertido em um embedding vetorial (representação matemática do problema) e armazenado no banco. Quando chega um novo chamado, o sistema busca automaticamente os casos mais similares já resolvidos — mesmo que o cliente use palavras diferentes para descrever o mesmo problema.

🔢

Busca Vetorial (Principal)

Converte a descrição do problema em 1.536 dimensões matemáticas e encontra os casos mais próximos por similaridade de cosseno. Funciona mesmo com linguagem informal ou incompleta.

🔤

Busca Textual (Fallback)

Se a busca vetorial não encontrar resultados suficientes, usa busca por palavras-chave com scoring léxico como backup. Garante resultados mesmo sem embedding.

07 · Mapa visual

Aba "Cérebro" — Visualização do Conhecimento

O sistema possui uma aba dedicada com um mapa visual interativo (grafo force-directed) que exibe os módulos de conhecimento e as conexões entre eles — similar ao grafo do Obsidian. Permite visualizar quais módulos estão mais relacionados, quais têm maior volume de chamados e navegar pelo conhecimento de forma intuitiva.

08 · Integrações

Plataformas e Integrações

O Mavo AI foi projetado para funcionar como um hub central que recebe mensagens de múltiplas plataformas. Qualquer sistema que consiga fazer uma requisição HTTP pode enviar mensagens para o Mavo AI e receber a resposta da IA.

Plataforma Tipo Status Canal
WillTalk WhatsApp Business Ativo WhatsApp
MTalk WhatsApp Business Ativo WhatsApp
API Direta v1 REST API Ativo Qualquer canal
Mavo Gestão Integração interna Em implantação Portal Web

Endpoints principais da API:

Endpoint Função
POST /api/orquestrador/v1/mensagem Processa uma mensagem do cliente e retorna a resposta da IA + estado atualizado da conversa
POST /api/ingestao/willtalk Webhook dedicado para eventos do WillTalk (mensagens, mídias, eventos de conversa)
POST /api/ingestao/mtalk Webhook dedicado para eventos do MTalk
GET /api/busca-semantica Busca casos similares na base de conhecimento para consulta manual
GET /api/metricas Retorna métricas consolidadas para o dashboard (volumes, taxas, SLA)
GET /api/cerebro/grafo Retorna os dados do grafo de conhecimento para visualização
GET /api/health Healthcheck do sistema (banco, IA, embedding)
09 · Segurança

Segurança e Multi-tenancy

Cada plataforma integrada é autenticada via API Key + identificador da organização (X-Tenant-ID). O sistema aplica rate limiting por plataforma para evitar abuso. Todos os dados de cada organização são isolados por organization_id — múltiplos clientes podem usar o mesmo sistema com total separação de dados.

Diferencias & Próximos Passos
12 · Diferenciais

O que torna o Mavo AI diferente

  • Especializado no AUGE ERP — não é um chatbot genérico. Conhece cada módulo, cada campo, cada tabela do banco, cada rejeição fiscal. O diagnóstico é cirúrgico.
  • Aprende com cada atendimento — cada caso resolvido vira conhecimento pesquisável. Quanto mais usa, mais preciso fica.
  • Contexto conversacional contínuo — lembra de tudo que foi dito na conversa, nunca repete perguntas já respondidas, detecta quando o cliente diz "funcionou mas surgiu outro problema".
  • Aceita imagens — o cliente tira um print do erro e envia no WhatsApp. A IA analisa a imagem, extrai a mensagem de erro e usa no diagnóstico.
  • Handoff com contexto total — quando transfere para humano, entrega um resumo técnico estruturado com todo o diagnóstico. O técnico não precisa perguntar nada de novo.
  • Multi-plataforma e multi-tenant — funciona com WillTalk, MTalk e qualquer outro sistema via API. Suporta múltiplos clientes/organizações no mesmo servidor.
  • Fora do horário comercial — atende normalmente às 2h da manhã, no final de semana, em feriado. Registra o chamado e resolve ou encaminha quando o técnico retornar.
